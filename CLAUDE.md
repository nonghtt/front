# CLAUDE.md

이 파일은 상담사 대시보드 프로토타입 프로젝트의 UI/UX 개발 가이드입니다.

## 프로젝트 목적

내부 검토를 위한 상담사 대시보드의 여러 화면 컨셉을 빠르게 탐색하고 프로토타입 구현하는 프로젝트입니다.

**주요 목표:**
- 여러 대시보드 레이아웃 컨셉 개발 및 비교
- 업무 효율성과 정보 계층 구조 검증
- 프로젝트 화면 기준 수립

## 빠른 시작

```bash
# 개발 환경 준비
npm install
npm run dev
```

브라우저가 자동으로 열리면 메인 대시보드가 표시됩니다.

## 레이아웃 구조

고정된 UI 구조로 메인 콘텐츠 영역에 집중합니다:

```
┌─ 헤더 (상단 고정) ────────────────────┐
│ 로고 | 소프트폰 | 로그아웃             │
├─┬─────────────────────────────────────┤
│Q│ 왼쪽 사이드바                       │
│M│ (메인 네비게이션, 토글 가능)        │
│║ │                                    │
│ │       메인 콘텐츠 영역              │
│ │    (대시보드 컨셉 개발 영역)        │
│ │                                    │
└─┴─────────────────────────────────────┘
```

- **헤더**: 상단 고정 (로고, 소프트폰, 로그아웃 버튼)
- **왼쪽 퀵 메뉴**: 왼쪽 가장자리 고정 (아이콘 메뉴, 데스크톱 전용)
- **왼쪽 사이드바**: 왼쪽 고정, 메뉴 버튼으로 토글 가능
- **메인 콘텐츠**: 나머지 공간 (프로토타입 개발 영역)

## 메인 페이지 컨셉 개발

### 컨셉 개발 가이드

**1단계: 새 컨셉 페이지 생성**
- `src/pages/MainPage.vue`를 템플릿으로 사용하거나 복사본 생성
- 컨셉명으로 파일명 지정 (예: `ConceptA.vue`, `ConceptB.vue`)

**2단계: 컨셉 디자인**
다음 필수 요소를 포함하세요:
- **민원인 정보 섹션**: 고객 프로필, 기본정보 표시, 기업별로 다양한 고객의 정보를 표출해야함(ui에서 많은 영역을 차지할 것으로 예상됨.) ex) 농업인이라면 해당 민원인이 갖고 있는 작물, 토지. 기업인이라면 회사, 회사부지, 회사의 종류 등등 
- **상담이력 섹션**: 통화 기록, 상담 내용 관리
- **대시보드 위젯**: 업무 관련 정보 (통계, 상태 등)

**3단계: 라우팅 추가**
라우터에 새 컨셉을 등록하여 전환 가능하게 합니다.

### 컨셉 평가 기준

각 컨셉을 평가할 때 다음을 고려하세요:

| 기준 | 확인 사항 |
|------|----------|
| **정보 접근성** | 주요 정보를 빠르게 찾을 수 있는가? |
| **시각적 계층** | 중요도에 따라 요소가 시각적으로 구분되는가? |
| **업무 효율성** | 상담사의 주요 업무 흐름이 자연스러운가? |
| **확장성** | 향후 새로운 정보 추가 시 대응 가능한가? |

## 디자인 시스템

### SCSS 디자인 토큰

`src/css/app.scss`에 정의된 변수를 사용하여 일관성 있는 스타일링:

- **색상**: `$flow-border`, `$flow-text-label`, `$flow-text-secondary` 등
- **레이아웃**: `$body-width` 등의 크기 변수
- **테두리**: `.a-border` 클래스와 수정자 (`.bdl-none`, `.bdr-none` 등)

### Quasar 컴포넌트

Quasar의 기본 컴포넌트를 활용하세요:
- 레이아웃: `QLayout`, `QDrawer`, `QHeader`, `QPageContainer`
- UI: `QBtn`, `QCard`, `QTable`, `QIcon` 등
- Material Icons 사용 가능

## 필수 UI 요소

### 1. 민원인 정보 섹션
기간계 연동 시스템의 고객 데이터 표시:
- 고객 이름, ID, 연락처
- 가입 상품 정보
- 계약 상태
- 기타 필수 정보

### 2. 상담이력 섹션
상담 기록 관리 및 표시:
- 상담 일시
- 상담 내용 요약
- 상담원 정보
- 다음 조치 사항
- 기록 저장 기능

### 3. 대시보드 위젯
업무 효율성을 위한 추가 요소:
- 대기 중인 고객 수
- 오늘의 상담 통계
- 미해결 사항 목록
- 빠른 접근 링크

## 컴포넌트 구조

프로토타입 개발을 위한 기본 구조:

```
src/
├── pages/
│   ├── MainPage.vue          # 기본 컨셉
│   ├── Concept*.vue          # 추가 컨셉들
│   └── ErrorNotFound.vue
├── components/
│   ├── FlowHeader.vue        # 헤더
│   ├── FlowQuickMenu.vue     # 왼쪽 퀵 메뉴
│   ├── MenuBtn.vue           # 메뉴 토글 버튼
│   └── [컨셉별 컴포넌트]
├── layouts/
│   └── MainLayout.vue        # 메인 레이아웃 (고정)
└── css/
    └── app.scss              # 디자인 토큰
```
